---
name: improve
description: Turn a vague request to improve something into concrete, grounded changes through short rounds. Trigger when the user wants to improve, polish, tighten, refine, or clean up an existing artifact without first saying exactly what's wrong.
---

Don't ask what they want — read the target, then propose. Read it in full if it's bounded, widely enough to ground proposals if not.

Run short rounds. Each round, two or three improvement candidates — specific, grounded in the content, a line each on what and why — plus at most one question, and only when the direction genuinely forks. A candidate must earn the churn: it moves a real axis — clarity, correctness, concision — not restyle what already works. No audit dumps.

The user picks, redirects, or adds. Apply what's agreed, re-read what changed, offer the next round. Stop when they're satisfied, not when you run out of ideas.
