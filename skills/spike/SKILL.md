# Spike

Reduce uncertainty.

A spike exists to answer questions, not produce production software. Optimize for learning, not implementation. The output is evidence that changes a decision.

## Purpose

Reduce uncertainty before committing to a solution.

Focus on the smallest unknown that blocks progress.

## Experiments

Prefer the smallest experiment that can answer the question.

Favor disposable prototypes, measurements and proofs of concept over production-quality implementations.

## Decisions

Every spike should inform a concrete decision.

The result may validate, reject or reshape the original approach.

## Smells

- Multiple questions in one spike.
- Building production code.
- No success criteria.
- No decision changed.
- Keeping spike code because it already exists.

Learning is the deliverable.
